## ⚡ 5-Minute Setup

### Step 1: Prepare Your Assets

1. **Create your logos** in these formats:
   - logo-primary.svg (full color)
   - logo-white.svg (for dark backgrounds)
   - logo-monochrome.svg (black only)
   - logo-icon.svg (icon mark)
   - logo-horizontal.svg (horizontal layout)

2. **Design your icons** - Create 24px × 24px SVG icons for:
   - analytics.svg
   - finance.svg
   - compliance.svg
   - corporate.svg
   - audit.svg
   - growth.svg
   - security.svg
   - process.svg

### Step 2: Replace Placeholder Assets

Replace the placeholder SVG files in `assets/` with your actual designs:
- `assets/logos/` - Your KPCA logos
- `assets/icons/` - Your icon set

### Step 3: Upload to Your Portal

1. Create folder: `Brand_Guidelines`
2. Upload entire folder contents:
   ```
   index.html
   styles.css
   README.md
   SETUP.md
   assets/
   ```

3. Link from your portal navigation to `index.html`

### Step 4: Test & Publish

- [ ] Test in Chrome, Firefox, Safari
- [ ] Check mobile view
- [ ] Test print functionality (Ctrl+P)
- [ ] Share link with team
- [ ] Create backup

## 🎨 Asset Specifications

### Logo Requirements
- **Format**: SVG (scalable, crisp)
- **Fallback**: PNG at 2x resolution if needed
- **Colors**: Use brand colors (#1E3A5F navy, #D4A574 gold)
- **File Size**: Keep SVGs under 50KB each

### Icon Requirements
- **Size**: 24px × 24px natural size
- **Format**: SVG with stroke style
- **Stroke Weight**: 2px
- **Style**: Simple, geometric, consistent
- **Colors**: #1E3A5F navy primary
- **Background**: Transparent

## 🚨 Common Issues & Fixes

**Q: Images not showing?**
- A: Check file paths match folder structure
- A: Use forward slashes: `assets/logos/logo.svg`
- A: Ensure SVG files actually exist

**Q: Styles look wrong?**
- A: Clear browser cache (Ctrl+Shift+Delete)
- A: Reload page (Ctrl+F5)
- A: Check styles.css is in same folder as index.html

**Q: Navigation links don't work?**
- A: Verify anchor IDs match href (#section matches id="section")
- A: Use lowercase IDs without spaces

**Q: Mobile view broken?**
- A: Check viewport meta tag exists
- A: Test in Chrome DevTools responsive mode

**Tip 5**: Export SVGs without compression for crisp display; add dimensions in viewBox
